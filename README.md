Текущая реализация загружает архив и файлы целиком в память.
Практически рекомендуется использовать файлы размером до нескольких сотен мегабайт.

Теоретический лимит определяется доступной оперативной памятью.

Текущая версия поддерживает большие файлы благодаря использованию uint64_t, однако пока использует memory-based модель архива. Следующим этапом планируется потоковая обработка данных при сжатии алгоритмом Хаффмана, что позволит работать с файлами практически любого размера.
