# ArchiveProject

Простейший архиватор на C++ с поддержкой собственного формата архива и алгоритма сжатия Хаффмана.

## Возможности

* Создание архивов
* Извлечение файлов
* Просмотр содержимого архива
* Получение информации об архиве
* Добавление файлов в существующий архив
* Удаление файлов из архива
* Поддержка вложенных папок
* CRC32 контроль целостности
* Сжатие алгоритмом Хаффмана
* Собственный формат архива
* Сборка через CMake


# Формат архива

Архив имеет собственный бинарный формат.

```text
[MAGIC]
[VERSION]
[FILE_COUNT]

[FILE_1_METADATA]
[FILE_2_METADATA]
...
[FILE_N_METADATA]

[COMPRESSED_DATA]
```

Метаданные файла:

```text
[nameLength]
[name]

[originalSize]
[compressedSize]

[dataOffset]

[crc32]

[frequencies[256]]
```

# Сборка

Создать директорию сборки:

```bash
mkdir build
cd build
```

Сгенерировать проект:

```bash
cmake ..
```

Собрать:

```bash
make
```

После успешной сборки появится исполняемый файл:

```bash
archive
```

# Использование

## Создание архива

```bash
./archive create test.arc test_files/a.txt test_files/b.txt
```

или

```bash
./archive create test.arc \
    test_files/a.txt \
    test_files/folder/c.txt
```

---

## Просмотр содержимого

```bash
./archive list test.arc
```

Пример:

```text
Files in archive (v2):

test_files/a.txt
test_files/b.txt
test_files/folder/c.txt
```

---

## Информация об архиве

```bash
./archive info test.arc
```

Пример:

```text
Archive v2 info

Files: 3

test_files/a.txt
  original size: 12
  compressed size: 6

test_files/b.txt
  original size: 12
  compressed size: 7

Total original size: 24
Archive size: 163
Compression ratio: 31.8%
```

---

## Извлечение архива

```bash
./archive extract test.arc output
```

Будет создана структура:

```text
output/
└── test_files/
    ├── a.txt
    ├── b.txt
    └── folder/
        └── c.txt
```

---

## Добавление файла

```bash
./archive add test.arc test_files/d.txt
```

После добавления:

```text
a.txt
b.txt
c.txt
d.txt
```

---

## Удаление файла

Можно удалить как по полному пути:

```bash
./archive remove test.arc test_files/a.txt
```

так и только по имени:

```bash
./archive remove test.arc a.txt
```

# Алгоритм сжатия

В проекте используется алгоритм Хаффмана.

Этапы сжатия:

1. Подсчёт частот символов.
2. Построение дерева Хаффмана.
3. Генерация кодов.
4. Побитовая упаковка данных.
5. Сохранение таблицы частот в архив.

Этапы распаковки:

1. Чтение таблицы частот.
2. Восстановление дерева.
3. Декодирование битового потока.
4. Восстановление исходного файла.

# Ограничения

Текущая реализация загружает архив и файлы целиком в память.

Практически рекомендуется использовать файлы размером до нескольких сотен мегабайт.

Теоретический лимит определяется доступной оперативной памятью.

Текущая версия поддерживает большие файлы благодаря использованию uint64_t, однако пока использует memory-based модель архива.

Следующим этапом планируется потоковая обработка данных при сжатии алгоритмом Хаффмана, что позволит работать с файлами практически любого размера.
